Master data

Master data usage

Master data is a collection of predefined options to create requests and test strategies, including:

  • Marker panels and trait marker dependencies for test strategists
  • Service types and budget numbers for customers
  • Field dependencies for customers and test strategists

Manage the master data

On the master data list screens you can perform the following actions:

Notice: Entities are retired to ensure traceability, otherwise, they are deleted.
Entity Can it be retired? Can it be deleted?
Trait marker dependency No Yes
Field dependency No Yes
Predefined panel Yes, but it cannot be reactivated No
Service type Yes, and it can be reactivated No, unless it was never used in a request

Set up predefined marker panels

Set up specific predefined marker panels for test strategists to select when creating a test strategy. These panels include frequently used marker instance combinations. For more information on predefined marker panels, see Testing strategy types.
  1. In the left blue menu, go to the Admin section and click Predefined panels.
  2. At the right top of the table, click > Create marker panel.
  3. In the Create marker panel window, enter the panel specifics, including a meaningful and unique name to easily identify and retrieve the panel. Click Create.
    Note: The marker instance unique identifiers (UIDs) can be found in the marker repository: https://marker-repository-app.rnd.seeds.basf.net/
    Notice: Next to routinely used panels, you can also include panels in the discovery phase, even if they haven’t been validated or ordered. When the panels do become available, check Available in the lab.

Set up predefined trait panels

Set up predefined trait panels for test strategists to select when creating a test strategy. These panels include frequently used trait combinations. For more information on predefined trait panels, see Testing strategy types.
  1. In the left blue menu, go to the Admin section and click Predefined panels.
  2. At the right top of the table, click > Create trait panel.
    The Create trait panel window appears, containing the trait marker dependencies table. This table helps you translate traits into the most preferred marker instances.
  3. Enter a meaningful and unique name to easily identify and retrieve the panel.
  4. Check the marker instances you want to combine in the panel. Click Create.
    Tip: You can filter the trait marker dependencies table on crop or technology.

Set up trait marker dependencies

Set up specific trait marker dependencies for test strategists to select when creating a test strategy. Trait marker dependencies combine two BASF sources to translate traits to the most preferred marker instance (that is the actual technique used to analyze). When a customer is interested in specific traits, the test strategist can translate this in technologies and instances based on the trait marker dependencies.
  1. In the left blue menu, go to the Admin section and click Trait marker dependencies.
  2. At the right top of the table, click > Create.
  3. In the Add trait marker dependency window, select the relevant crop and enter traits on two levels:
    • On cluster level (for example diseases)
    • On sub-cluster level
  4. Select the corresponding marker instances.
    Tip: Click to view all options of a specific field maintained in a central ontology. Search the ontology by entering a search term at the top of the list.
  5. Optionally, add remarks.
    Tip: You can include trait marker dependencies before they are available. Simply create the dependency and check the Available in the lab box once available.
  6. Click Create.

Set up service types

Set up specific types of services for customers to select when creating a request.
  1. In the left blue menu, go to the Admin section and click Service types.
  2. At the right top of the table, click > Create.
  3. On the Main tab, enter general service type info, including meaningful analysis type names and a description to avoid confusion among customers.
    Notice: Check the Requires testing strategy box, unless the service type doesn't need a testing strategy, such as DNA isolation or a plate search from the inventory.
  4. On the Additional info tab, specify additional types of information the test strategist must provide and click Save.
    Note: To prevent test strategists from defining a strategy without adding a particular type of additional info, click to add the info type and check the mandatory box.
    Tip: Make sure to clearly specify the kind of information you expect from the customer.

Set up budget numbers

Set up specific budget numbers for downstream invoicing for customers to select when creating a request.
Important: Budget numbers can only be defined for the business unit Vegetable Seeds. For Field Crops, a central database is maintained.
  1. In the left blue menu, go to the Admin section and click Budget numbers.
  2. At the right top of the table, click .
  3. Upload a TimeTell file and click Save.
    The codes and descriptions in your file will be placed in the list. New entries are added to the list and updated descriptions are updated in the list.

Set up field dependencies

Field dependencies help reduce the complexity of the MARqeT system and prevent users from clicking invalid options. In order to obtain this, field dependencies link users to specific field options only valid for that group of users. This results in reduced options, as some of them become unselectable.
Attention: Carefully consider the field dependencies you define. You don't want to mistakenly exclude too many options.
  1. In the left blue menu, go to the Admin section and click Field dependencies.
  2. At the right top of the table, click > Create.
  3. In the Add field dependency window, select the relevant business unit.
  4. For each required dependency:
    1. Click Add field and select the field type.
    2. Click and check the relevant values.
    Important: Not all service types and testing strategies are available for all crops.
  5. Click Create.
